# Fine-Tune SLM: Code Comment Generation

Fine-tuning **Gemma 3 270M** with LoRA adapters to generate docstrings/comments for code snippets, trained and evaluated entirely on an M1 MacBook Air (8 GB RAM, CPU-only).

## Pipeline

Run all steps in order, or use the convenience script:

```bash
bash run_pipeline.sh
```

| Step | Script | What it does |
|------|--------|--------------|
| 0 | `0_data_prep.py` | Load & clean the parquet file, split 70/15/15 into train/eval/test |
| 1 | `1_baseline_benchmark.py` | Evaluate untuned Gemma 3 270M on 50 eval examples (BLEU-4, ROUGE-L) |
| 2 | `2_finetune.py` | Fine-tune with LoRA (r=16) on 500 training examples for 3 epochs |
| 3 | `3_evaluate_finetuned.py` | Evaluate fine-tuned model on the same 50 examples |
| 4 | `4_comparative_analysis.py` | Generate comparison report, CSV table, and bar chart |

---

## Approach

- **Model**: `google/gemma-3-270m-it`
- **Method**: LoRA (Low-Rank Adaptation) targeting `q_proj` and `v_proj` attention layers — only ~1–2M of 270M parameters are trained
- **Dataset**: [CodeSearchNet](https://github.com/github/CodeSearchNet) — `comment` / `code` column pairs
- **Metrics**: BLEU-4 (n-gram overlap) and ROUGE-L (longest common subsequence)
- **Hardware**: M1 MacBook Air, 8 GB RAM, CPU-only (`adamw_torch`, `float32`)

## Note on Missing Files

> **The dataset, model checkpoints, and LoRA adapter are not included in this repository** due to file size constraints (GitHub's 100 MB per-file limit and repository size cap).
>
> - **Dataset**: Download `train-00000-of-00003.parquet` from the [CodeSearchNet dataset on HuggingFace](https://huggingface.co/datasets/code-search-net/CodeSearchNet) and place it in `data/`.
> - **Model weights**: Gemma 3 270M is downloaded automatically from HuggingFace the first time a script runs (`google/gemma-3-270m-it`). A HuggingFace account and accepted model licence are required.
> - **LoRA adapter** (`models/lora_adapter/`): Generated by running `2_finetune.py`.
> - **Processed splits** (`data/processed/`): Generated by running `0_data_prep.py`.
